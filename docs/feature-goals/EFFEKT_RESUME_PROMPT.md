# Prompt: Återstarta Effekt-kapitel uppdateringar

## Använd denna prompt när du vill återstarta arbetet:

```
Jag vill att du fortsätter med att uppdatera Effekt-kapitlen i feature goal HTML-filerna enligt den arbetsprocess vi etablerat.

**KONTEXT:**
- Vi har uppdaterat Effekt-kapitlen för följande filer (klar):
  1. ✅ mortgage-application-v2.html
  2. ✅ mortgage-appeal-v2.html
  3. ✅ mortgage-offer-v2.html
  4. ✅ mortgage-kyc-v2.html

- Återstående filer finns listade i: `docs/feature-goals/EFFEKT_UPDATE_LIST.md`

- Arbetsprocessen och riktlinjerna finns i:
  - `docs/feature-goals/MANUAL_HTML_WORKFLOW.md` (sektion om "Effekt")
  - `docs/feature-goals/EFFEKT_EXECUTIVE_SUMMARY_EXEMPEL.md`
  - `docs/feature-goals/EFFEKT_AGGREGATION_ANALYSIS.md`

**STRUKTUR FÖR EFFEKT-KAPITLET (OBLIGATORISKT):**
1. Executive Summary (direktörsvänlig, kortfattad, 3-4 kategorier)
2. Detaljerade sektioner med tabeller (OBLIGATORISKT för alla beräkningar)
3. Jämförelse med nuvarande process (tabell)
4. Aggregeringsinformation (OBLIGATORISKT - tabell med kolumner: Effekt, Typ, Volym, Aggregeringsbar, Redan inkluderad i parent)

**VIKTIGT:**
- Använd ALLTID tabeller för detaljerade beräkningar (sektion 1, 2, 3)
- Lägg ALLTID till Aggregeringsinformation-sektion
- Följ strukturen från de redan uppdaterade filerna som referens
- Uppdatera `docs/feature-goals/EFFEKT_UPDATE_LIST.md` när en fil är klar

**DIN UPPGIFT:**
Fortsätt med nästa fil i listan (`docs/feature-goals/EFFEKT_UPDATE_LIST.md`) och uppdatera dess Effekt-kapitel enligt arbetsprocessen. Gå igenom filerna en efter en tills alla är klara.
```

## Ytterligare instruktioner (om behövs):

Om du vill fortsätta från en specifik fil:
```
Fortsätt med att uppdatera Effekt-kapitlet för [FILNAMN].html enligt arbetsprocessen.
```

Om du vill se status:
```
Visa mig status för Effekt-kapitel uppdateringar - vilka filer är klara och vilka återstår?
```

Om du vill testa aggregering igen:
```
Testa aggregering av effektdata från de uppdaterade filerna för att verifiera att processen fungerar.
```
