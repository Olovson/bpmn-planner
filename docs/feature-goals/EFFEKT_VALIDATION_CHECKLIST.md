# Checklista: Validering av Effekt-kapitlet

## Executive Summary

- [ ] Max 1-2 paragrafstycken eller kort lista med 5-7 nyckeltal
- [ ] Fokus på affärsmässiga effekter (tidssparande, kostnader, kapacitet, kundupplevelse)
- [ ] Inga tekniska detaljer (ingen BPMN-terminologi, inga processsteg)
- [ ] Lätt att skanna (bullet points eller kortfattade meningar)
- [ ] Konkreta siffror i fetstil för att framhäva nyckeltal
- [ ] Tydlig struktur med kategorier (Kostnadsbesparingar, Kapacitetsökning, Kundupplevelse)
- [ ] Volym-information inkluderad (t.ex. "baserat på 100 000 ansökningar per år")
- [ ] Förtydligande av relationen mellan olika beräkningar (t.ex. "22 FTE elimineras genom direkt automatisering" vs "37.5% personalbesparing - total kapacitetsökning inklusive parallellisering")
- [ ] Notering om konservativa uppskattningar och saknad baseline-data

## Detaljerade sektioner

### Sektion 1: Automatisering och kostnadsbesparingar

- [ ] Förklarar hur man kom fram till siffrorna i Executive Summary
- [ ] Kort textbeskrivning av BPMN-mekanismen (1-2 meningar)
- [ ] Tabell för beräkningar (OBLIGATORISKT) med kolumner: Aktivitet, Volym, Tid per ansökan, Total tidssparande/år, FTE-värde
- [ ] Text för processförbättringar och noteringar
- [ ] Innehåller tekniska detaljer (BPMN-mekanismer, processsteg)
- [ ] Visar beräkningsmetodik (volym × tid = total tidssparande)

### Sektion 2: Snabbare processering och förbättrad kundupplevelse

- [ ] Förklarar hur man kom fram till siffrorna i Executive Summary
- [ ] Kort textbeskrivning av BPMN-mekanismen (1-2 meningar)
- [ ] Tabell för beräkningar (OBLIGATORISKT) med kolumner: Aspekt, Nuvarande, Nytt system, Förbättring
- [ ] Text för processförbättringar och noteringar
- [ ] Innehåller tekniska detaljer (BPMN-mekanismer, processsteg)
- [ ] Visar beräkningsmetodik

### Sektion 3: Kapacitetsökning

- [ ] Förklarar hur man kom fram till siffrorna i Executive Summary
- [ ] Kort textbeskrivning (1-2 meningar)
- [ ] Tabell för beräkningar (OBLIGATORISKT) med kolumner: Aspekt, Nuvarande, Nytt system, Förbättring
- [ ] Förtydligande av relationen mellan 22 FTE (direkt automatisering) och 37.5% personalbesparing (total kapacitetsökning)
- [ ] Text för processförbättringar och noteringar
- [ ] Innehåller tekniska detaljer (BPMN-mekanismer, processsteg)
- [ ] Visar beräkningsmetodik

## Jämförelse med nuvarande process

- [ ] Tabell med kolumner: Aspekt, Nuvarande, Nytt system, Förbättring
- [ ] Inkluderar "Ansökningar som når handläggare" om relevant
- [ ] Konsekventa siffror med resten av kapitlet

## Aggregeringsinformation (OBLIGATORISKT)

- [ ] Tabell med kolumner: Effekt, Typ, Volym, Aggregeringsbar, Redan inkluderad i parent
- [ ] Markerar direkta effekter (kan aggregeras) vs indirekta effekter (redan inkluderade)
- [ ] Exakt volym för varje effekt
- [ ] Förtydligar om effekten redan är inkluderad i huvudprocessen

## Konsekvens och kvalitet

- [ ] Konsekventa siffror genom hela kapitlet (samma siffror för samma sak)
- [ ] Tydliga beräkningar (visar uppdelning per aktivitet)
- [ ] Tolkningsbara siffror (t.ex. "100 000 ansökningar × 1 dag" istället för "100 000 dagar")
- [ ] Separerade olika typer av avslag (pre-screening vs totalt avslag)
- [ ] Förtydligad relation mellan olika beräkningar (22 FTE vs 37.5% personalbesparing)
