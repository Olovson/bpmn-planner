Feature Goal

Internal data gathering

  • Initiativ: mortgage-se-application
  • BPMN Call Activity: internal-data-gathering (Internal data gathering)
  • Regel/affärsägare: Produktägare Kredit / Risk & Policy
  • Kreditprocess-steg: mortgage-se-application
  • Version / datum: 1.0 (LLM-validerad) – uppdateras vid ändring

Sammanfattning & scope

Internal data gathering syftar till att effektivisera insamlingen av kund- och engagemangsdata för att förbättra kreditbedömningen. Denna funktion riktar sig mot privatkunder som ansöker om bolån och stödjer bankens strategi för att minska kreditrisker och förbättra kundupplevelsen genom snabbare och mer precisa beslut.

  • Ingår: Ingår: insamling av intern kund- och engagemangsdata för kreditbedömning.; Ingår: automatiserad validering av insamlad data mot befintliga kundregister.; Ingår: integration med interna system för att hämta uppdaterad information om kundens ekonomiska situation.; Ingår: generering av rapporter för riskanalys baserat på insamlad data.
  • Ingår inte: Ingår inte: eftermarknadsprocesser och generella engagemangsändringar.; Ingår inte: manuell datajustering av handläggare.

Effektmål

  • Automatisera insamlingen av relevant kunddata för att minska manuellt arbete och ledtider.
  • Förbättra kvaliteten och säkerheten i kreditbedömningar genom tillgång till mer omfattande data.
  • Förbättra kundupplevelsen genom att ge snabbare och tydligare besked om kreditbeslut.
  • Stärka regelefterlevnad och riskkontroll genom att säkerställa korrekt och fullständig datahantering.

Ingående Epics

Epic-ID Epic-namn Beskrivning Team
E1 Data Collection Automation Automatisera insamlingen av kunddata från interna system för att minska manuellt arbete. Risk & Kredit
E2 Data Validation and Integration Validera och integrera insamlad data med befintliga kundregister för att säkerställa datakvalitet. IT & Systemutveckling
E3 Risk Analysis Reporting Generera rapporter för riskanalys baserat på den insamlade och validerade datan. Analys & Rapportering

Affärsflöde

  1. Kunden initierar en bolåneansökan via bankens webbportal.
  2. Systemet samlar automatiskt in relevant kund- och engagemangsdata från interna system.
  3. Den insamlade datan valideras mot befintliga kundregister för att säkerställa korrekthet.
  4. Rapporter genereras för att stödja riskanalys och kreditbeslut.
  5. Kreditbeslutet kommuniceras till kunden via bankens digitala kanaler.

Affärs-scenarion

Scenario Beskrivning Typ (Happy/Edge/Error) Förväntat resultat
S1 Successful Data Gathering Happy All relevant data samlas in och valideras framgångsrikt, vilket leder till ett preliminärt godkännande av kreditansökan.
S2 Data Validation Failure Edge Viss insamlad data kan inte valideras, vilket leder till att ansökan skickas till manuell granskning.
S3 System Integration Error Error Ett integrationsfel uppstår, vilket förhindrar insamling av viss data och kräver teknisk support.
