Feature Goal

Internal data gathering

  • Initiativ: mortgage-se-application

  • Regel/affärsägare: Produktägare Kredit / Risk & Policy
  • Kreditprocess-steg: mortgage-se-application
  • Version / datum: 1.0 (LLM-validerad) – uppdateras vid ändring

Sammanfattning & scope

Målet med denna funktion är att effektivisera insamlingen av intern data för kreditbedömning inom bolåneprocessen. Detta omfattar kunder som ansöker om bolån och syftar till att förbättra kvaliteten och hastigheten i kreditbeslutsprocessen. Genom att automatisera datainsamlingen stödjer vi bankens strategi för riskhantering och förbättrar kundupplevelsen genom snabbare och mer precisa beslut.

  • Ingår: Ingår: insamling av intern kund- och engagemangsdata för kreditbedömning.; Ingår: automatiserad validering av insamlad data mot interna databaser.; Ingår: integration med interna system för att hämta nödvändig information.; Ingår: uppdatering av kreditbeslutsmotor med insamlad data.
  • Ingår inte: Ingår inte: eftermarknadsprocesser och generella engagemangsändringar.; Ingår inte: insamling av extern data från tredje part.

Effektmål

  • Automatisering av datainsamling minskar manuellt arbete och kortar ledtider i kreditbedömningen.
  • Förbättrad kvalitet och säkerhet i kreditbedömningar genom tillgång till aktuell och korrekt intern data.
  • Förbättrad kundupplevelse genom snabbare och tydligare besked om kreditbeslut.
  • Stärkt regelefterlevnad och riskkontroll genom att säkerställa att all relevant data beaktas i beslutsprocessen.

Ingående Epics

Epic-ID Epic-namn Beskrivning Team
E1 Data Integration Integrera interna system för att möjliggöra effektiv datainsamling. IT & Infrastruktur
E2 Data Validation Automatisera validering av insamlad data för att säkerställa korrekthet. Risk & Kredit
E3 Decision Update Uppdatera kreditbeslutsmotor med ny data för att förbättra beslutsprocessen. Produktutveckling

Affärsflöde

  1. Kunden skickar in en ansökan om bolån via bankens webbportal.
  2. Systemet initierar insamling av intern data från relevanta databaser.
  3. Den insamlade datan valideras automatiskt mot interna regler och kriterier.
  4. Validerad data integreras i kreditbeslutsmotorn för vidare bearbetning.
  5. Kreditbeslutet uppdateras och kommuniceras till kunden via webbportalen.

Affärs-scenarion

Scenario Beskrivning Typ (Happy/Edge/Error) Förväntat resultat
S1 Successful Data Collection Happy All nödvändig data samlas in och valideras framgångsrikt, vilket leder till ett preliminärt godkännande.
S2 Data Validation Error Error Valideringsfel upptäcks i insamlad data, vilket kräver manuell granskning.
S3 Incomplete Data Edge Viss data saknas, vilket leder till att ansökan skickas tillbaka till kunden för komplettering.
