Feature Goal

Internal data gathering

  • Initiativ: mortgage-se-application
  • BPMN Call Activity: internal-data-gathering (Internal data gathering)
  • Regel/affärsägare: Produktägare Kredit / Risk & Policy
  • Kreditprocess-steg: mortgage-se-application
  • Version / datum: 1.0 (LLM-validerad) – uppdateras vid ändring

Sammanfattning & scope

Detta Feature Goal möjliggör effektiv och automatiserad intern datainsamling under kreditprocessens datainsamlingsfas. Regelmotorn spelar en central roll i att samla in och bearbeta nödvändig information för att stödja en korrekt och snabb kreditbedömning. Genom att fokusera på interna data säkerställs att banken kan fatta välgrundade beslut baserade på tillförlitlig information, vilket stärker bankens kreditstrategi och riskhantering.

Ingår

  • Automatiserad insamling av interna data för kreditbedömning.
  • Användning av regelmotor för att bearbeta insamlad data.
  • Integration med interna datakällor för att säkerställa datakvalitet.
  • Stöd för att fatta preliminära kreditbeslut baserat på insamlad data.

Ingår inte

  • Manuell datainsamling från externa källor.
  • Slutgiltiga kreditbeslut som kräver ytterligare manuell granskning.

Effektmål

  • Automatisering av datainsamling minskar manuellt arbete och kortar ledtider.
  • Förbättrad kvalitet och säkerhet i kreditbedömningar genom tillgång till korrekt och aktuell data.
  • Förbättrad kundupplevelse genom snabbare och mer precisa besked.
  • Stärkt regelefterlevnad och riskkontroll genom systematiserad datainsamling.

Affärsflöde

  1. Systemet initierar datainsamling från interna källor.
  2. Regelmotorn bearbetar den insamlade datan för att säkerställa kvalitet.
  3. Systemet genererar en preliminär kreditbedömning baserat på tillgänglig data.
  4. Resultatet av datainsamlingen skickas vidare för ytterligare analys vid behov.

Affärs-scenarion

Scenario Beskrivning Typ (Happy/Edge/Error) Förväntat resultat
S1 Successful Data Collection Happy Data samlas in och bearbetas framgångsrikt, vilket leder till en preliminär kreditbedömning.
S2 Partial Data Availability Edge Endast delvis data är tillgänglig, vilket kräver manuell komplettering för fullständig bedömning.
S3 Data Source Unavailable Error En eller flera datakällor är otillgängliga, vilket förhindrar fullständig datainsamling.
S4 Rule Engine Failure Error Regelmotorn misslyckas med att bearbeta data, vilket kräver felsökning och manuell intervention.
S5 Data Quality Issue Edge Data innehåller felaktigheter som kräver manuell granskning och korrigering.
S6 Successful Integration Happy Integration med alla datakällor fungerar smidigt, vilket möjliggör en snabb och korrekt bedömning.

Implementation Notes (för dev)

  • Använd RESTful API-mönster för att integrera med interna datakällor.
  • Loggning och audit-spår ska implementeras för att säkerställa regelefterlevnad.
  • Felhantering bör inkludera återförsök och notifieringar vid misslyckade datainsamlingar.
  • Prestandaoptimering krävs för att hantera stora datamängder effektivt.

Tekniska & externa beroenden

  • Regelmotor(er) och beslutsmotorer kopplade till Feature Goalet.
  • Datakällor (interna register, databaser och domäntjänster) som används i flödet.
  • Externa API:er och tjänster som tillhandahåller kredit-, person- eller objektsdata.
  • Integrationer och integrationslager som hanterar kommunikation mot interna/externa system.
  • Påverkade interna system, moduler och komponenter som behöver anpassas eller övervakas.

Relaterade regler / subprocesser

  • Feature Goal: Extern datainsamling för att komplettera interna data.
  • Epic: Data Quality Assurance för att säkerställa korrekthet i insamlad data.
  • Business Rule: Kreditvärdighetsbedömning baserad på insamlad data.
