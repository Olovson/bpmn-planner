Sammanfattning & scope

Feature Goalet för intern datainsamling syftar till att effektivisera och automatisera insamlingen av kunddata för kreditbedömning. Det omfattar kunder som ansöker om bolån och stödjer bankens strategi för att förbättra riskhantering och kundupplevelse genom att säkerställa att all relevant information är tillgänglig och korrekt från början.

  • Ingår: Ingår: insamling av intern kund- och engagemangsdata för kreditbedömning.; Ingår: automatiserad validering av insamlad data mot interna och externa källor.; Ingår: integration med befintliga system för att hämta nödvändig data.; Ingår: stöd för att identifiera och flagga avvikande data för manuell granskning.
  • Ingår inte: Ingår inte: eftermarknadsprocesser och generella engagemangsändringar.; Ingår inte: hantering av kundklagomål och tvister.

Effektmål

  • Automatisering av datainsamling minskar manuellt arbete och kortar ledtider i kreditprocessen.
  • Förbättrad kvalitet och säkerhet i kreditbedömningar genom tillgång till korrekt och aktuell data.
  • Förbättrad kundupplevelse genom snabbare och mer precisa besked.
  • Stärkt regelefterlevnad och riskkontroll genom att säkerställa att all nödvändig data samlas in och granskas.

Ingående Epics

Epic-ID Epic-namn Beskrivning Team
E1 Automatisk datainsamling Automatisera insamlingen av kunddata från interna system för att minimera manuellt arbete. Data & Integration
E2 Data validering Validera insamlad data mot interna och externa källor för att säkerställa korrekthet och fullständighet. Risk & Kredit
E3 Avvikelsehantering Identifiera och flagga avvikande data för manuell granskning och åtgärd. Compliance

Affärsflöde

  1. Kunden påbörjar ansökan om bolån och lämnar grundläggande information.
  2. Systemet samlar automatiskt in kompletterande data från interna system.
  3. Insamlad data valideras mot interna och externa källor för att säkerställa korrekthet.
  4. Eventuella avvikelser flaggas för manuell granskning av handläggare.
  5. Slutlig kreditbedömning görs baserat på den validerade datan.

Affärs-scenarion

Scenario Beskrivning Typ (Happy/Edge/Error) Förväntat resultat
S1 Fullständig data insamlad Happy All nödvändig data samlas in och valideras utan avvikelser, vilket leder till en snabb och smidig kreditbedömning.
S2 Ofullständig data Edge Viss data saknas, vilket leder till att ansökan flaggas för manuell granskning.
S3 Avvikande data Error Avvikelser upptäcks i den insamlade datan, vilket kräver ytterligare utredning av handläggare.
