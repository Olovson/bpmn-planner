Sammanfattning & scope

Feature Goalet för intern datainsamling syftar till att effektivisera och automatisera insamlingen av kund- och engagemangsdata för kreditbedömning. Det omfattar alla kunder som ansöker om bolån och stödjer bankens strategi för att förbättra riskhantering och kundupplevelse genom snabbare och mer träffsäkra kreditbeslut.

  • Ingår: Ingår: insamling av intern kund- och engagemangsdata för kreditbedömning.; Ingår: automatiserad kontroll av befintliga kundrelationer och riskflaggor.; Ingår: integration med interna system för att hämta uppdaterad kundinformation.; Ingår: stöd för att hantera och lagra insamlad data på ett säkert sätt.
  • Ingår inte: Ingår inte: eftermarknadsprocesser och generella engagemangsändringar.; Ingår inte: manuell datainsamling och bearbetning utanför systemet.

Effektmål

  • Automatisering av datainsamling minskar manuellt arbete och kortar ledtider för kreditbeslut.
  • Förbättrad kvalitet i kreditbedömningar genom mer fullständig och korrekt data.
  • Ökad kundnöjdhet genom snabbare besked och färre omtag i ansökningsprocessen.
  • Stärkt regelefterlevnad genom att säkerställa att all relevant data samlas in och granskas.

Ingående Epics

Epic-ID Epic-namn Beskrivning Team
E1 Automatiserad kunddatainsamling Denna epic fokuserar på att automatisera insamlingen av kunddata från interna system. Data & Analys
E2 Riskflaggsanalys Denna epic hanterar analys och identifiering av riskflaggor baserat på insamlad data. Risk & Kredit

Affärsflöde

  1. Kunden ansöker om bolån via bankens digitala plattform.
  2. Systemet initierar insamling av kunddata från interna system.
  3. Insamlad data analyseras för att identifiera eventuella riskflaggor.
  4. Resultaten från analysen används för att fatta preliminära kreditbeslut.

Affärs-scenarion

Scenario Beskrivning Typ (Happy/Edge/Error) Förväntat resultat
S1 Standardansökan Happy Kunddata samlas in och analyseras utan problem, vilket leder till ett preliminärt godkännande.
S2 Riskflagga identifierad Edge En riskflagga identifieras och ansökan skickas till manuell granskning.
S3 Systemfel vid datainsamling Error Datainsamlingen misslyckas och ansökan pausas tills problemet är löst.

Implementation Notes (för dev)

  • Använd RESTful API-mönster för integration med interna system.
  • Säkerställ att all insamlad data loggas och auditeras enligt bankens säkerhetspolicy.
  • Implementera robust felhantering för att hantera eventuella systemfel under datainsamlingen.
  • Optimera prestanda för datainsamling för att minimera svarstider och maximera användarupplevelsen.
